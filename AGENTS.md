# AGENTS.md

## Frontend design rules

For every frontend, UI, landing page, animation, component, layout, or styling task:

- Use the `emil-design-eng` skill when available.
- Use `review-animations` after implementing motion when available.
- Treat visual taste as part of the task, not as optional polish.
- Do not produce generic AI-looking UI.
- Avoid random gradients, weak glassmorphism, template cards, boring centered hero sections, and unmotivated animations.
- Prioritize strong hierarchy, spacing rhythm, typography, interaction quality, and purposeful motion.
- Animations must be subtle, fast, interruptible, performant, and meaningful.
- Prefer transform and opacity for animation.
- Respect accessibility and reduced-motion preferences.
- Keep performance clean: avoid expensive layout thrashing, unnecessary re-renders, and heavy animation patterns.

Before editing UI:
1. Inspect the current structure and styling.
2. Identify what feels weak, generic, or amateur.
3. Decide what would make it feel more premium.
4. Then implement.

Before final response:
1. Review layout, spacing, typography, motion, accessibility, and performance.
2. Fix anything that still feels generic.
3. Summarize what changed and why.

## Taste bar

The final result should feel like it was made by a strong design engineer, not like a default SaaS template.

Aim for:
- premium but not overdesigned
- modern but not trendy-trash
- motion with restraint
- clear visual hierarchy
- confident whitespace
- sharp details
- tasteful interactions
- polished empty states, loading states, hover states, and transitions
